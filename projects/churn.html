Overview

Beta Bank is experiencing steady customer churn. Retaining customers is more cost-effective than acquiring new ones. This project develops and evaluates predictive models to identify customers at high risk of leaving, enabling targeted retention interventions.

Business Problem

  • Identify likely churners before account closure
  • Reduce revenue loss from attrition
  • Balance intervention cost against false positives
  • Select a model aligned with real business tradeoffs

Note: Because churn is a minority class, accuracy alone can be misleading.

Why This Matters

If churn risk is detected early, retention incentives can be targeted efficiently, high-value customers can be prioritized, and long-term customer lifetime value improves. This project focuses on decision-support metrics rather than simply maximizing accuracy.

Data & Preparation

  • Loaded and validated structured customer data (demographics, account behavior, tenure, balance)
  • Removed non-predictive identifiers (RowNumber, CustomerId, Surname)
  • Addressed missing values in Tenure
  • Encoded categorical features (Gender, Geography)
  • Performed EDA to understand churn distribution and class imbalance

Modeling Approach

  • Baseline: Logistic Regression (interpretability)
  • Compared: Decision Tree, Random Forest
  • Used train/test split and cross-validation
  • Tuned hyperparameters
  • Evaluated using ROC-AUC and F1-score to reflect precision/recall tradeoffs

Results

Rather than selecting a model based solely on accuracy, evaluation focused on metrics aligned with churn's class imbalance.

  • Compared Logistic Regression, Decision Tree, and Random Forest models
  • Evaluated performance using ROC-AUC and F1-score
  • Analyzed precision-recall tradeoffs
  • Reviewed threshold effects to balance missed churners vs unnecessary outreach

Final model: improved F1 and ROC-AUC relative to baseline and produced churn risk probabilities that can be used to rank customers for retention outreach.

Business Interpretation

Churn modeling is a cost-balancing exercise. False negatives (missed churners) risk lost revenue; false positives increase incentive spend. By evaluating tradeoffs instead of relying on a default cutoff, the model supports retention decisions aligned with financial priorities.

What This Demonstrates

  • End-to-end supervised learning workflow (EDA → modeling → evaluation → decision framing)
  • Handling imbalanced classification problems
  • Model comparison and evaluation discipline
  • Business-aligned metric selection
  • Clear communication of tradeoffs
